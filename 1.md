## 코틀린의 기본형
원시형을 없애고 기본형을 사용함
- 코드에서는 무조건 wrapper -> 컴파일때는 해당 플랫폼에서 유리한 것으로 변환
- 불변이나 아니냐에만 관심을 가짐 (불변은 복사듬 뭐든 알빠 없다는 마인드)

## 코틀린의 내장형
- Any : 모든 형의 부모
- Nothing : 모든 형의 자식 / 제어의 종결 -> 예외
- Unit : 없을 을 나타내는 형 프로시져를 없애기 위해 (void는 형이아니라 마킹 js의 undefined 와 같음)
- Throwable : 모든 예외의 부모
- Function : 함수를 나타냄
- KClass, KCallable - 리플렉션 타입
